## 최초 프롬프트
HTML, CSS, JavaScript를 사용해서 미니 스프레드시트 웹앱을 제작해 주세요.

이 웹앱은 JavaScript 강의에서 배운 내용을 활용하여 만드는 과제용 프로젝트입니다. 단순히 표 모양만 보여주는 것이 아니라, 사용자가 셀에 직접 데이터를 입력하고, 현재 선택된 셀의 위치를 확인하며, 입력된 전체 데이터를 Excel 파일로 Export할 수 있어야 합니다.

프로젝트는 반드시 다음 3개의 파일로 분리해서 작성해 주세요.

1. index.html
2. style.css
3. app.js

전체 요구사항은 다음과 같습니다.

1. 스프레드시트 그리드 생성

- 열은 A부터 I까지 총 9개를 생성해 주세요.
- 행은 1부터 9까지 총 9개를 생성해 주세요.
- 위쪽에는 A~I 열 헤더가 보여야 합니다.
- 왼쪽에는 1~9 행 헤더가 보여야 합니다.
- 각 셀은 사용자가 직접 텍스트, 숫자, 한글을 입력할 수 있는 input 형태로 만들어 주세요.
- 페이지가 로드되면 JavaScript로 동적으로 그리드를 생성해 주세요.

2. 현재 선택된 셀 좌표 표시

- 사용자가 셀을 클릭하거나 입력하면 현재 focus 된 셀의 좌표를 화면 상단에 표시해 주세요.
- 좌표 형식은 `Cell: D4`처럼 열 문자와 행 번호를 조합해서 표시해 주세요.
- 예를 들어 C열 1행을 선택하면 `Cell: C1`이 표시되어야 합니다.
- D열 4행을 선택하면 `Cell: D4`가 표시되어야 합니다.

3. 행/열 헤더 하이라이트

- 현재 focus 된 셀의 위쪽 열 헤더와 왼쪽 행 헤더가 함께 하늘색으로 하이라이트되게 해 주세요.
- 예를 들어 D4 셀을 선택하면 D 열 헤더와 4 행 헤더가 동시에 하늘색으로 표시되어야 합니다.
- 새로운 셀을 선택하면 이전 하이라이트는 제거되고, 새로 선택된 셀의 행/열 헤더만 하이라이트되어야 합니다.
- 현재 선택된 셀 자체도 테두리나 배경색으로 구분되게 해 주세요.

4. 셀 데이터 입력 및 유지

- 각 셀은 독립적인 값을 가져야 합니다.
- 사용자가 A2에 `3333`, B3에 `222`, D4에 `ㅇ의` 같은 값을 입력할 수 있어야 합니다.
- 다른 셀을 클릭하거나 이동해도 입력한 값은 유지되어야 합니다.
- 한글, 영어, 숫자 입력이 모두 가능해야 합니다.

5. 전체 셀 데이터 수집

- 입력된 모든 셀 데이터를 JavaScript에서 2차원 배열 형태로 수집할 수 있게 해 주세요.
- 행과 열의 위치가 실제 스프레드시트 좌표와 일치해야 합니다.
- 비어 있는 셀은 빈 문자열 `""`로 저장해 주세요.
- 예를 들어 A2에 값이 있으면 2차원 배열의 두 번째 행, 첫 번째 열 위치에 저장되어야 합니다.

6. Excel 파일 Export

- 화면 상단 또는 그리드 근처에 `Export SpreadSheet` 버튼을 만들어 주세요.
- 사용자가 이 버튼을 누르면 현재 입력된 모든 셀 데이터를 Excel `.xlsx` 파일로 다운로드할 수 있어야 합니다.
- 파일명은 `mini-spreadsheet.xlsx`로 지정해 주세요.
- Excel 파일 생성에는 SheetJS 라이브러리를 사용해 주세요.
- `index.html`에는 SheetJS CDN을 포함해 주세요.

사용할 CDN은 다음과 같습니다.

<script src="https://cdn.jsdelivr.net/npm/xlsx/dist/xlsx.full.min.js"></script>

7. Google Spreadsheet Import 호환성

- 생성된 Excel 파일을 Google Spreadsheet에서 Import했을 때도 웹앱에서 입력한 데이터가 같은 셀 위치에 나타나야 합니다.
- A1에 입력한 값은 Google Spreadsheet에서도 A1에 보여야 합니다.
- D4에 입력한 값은 Google Spreadsheet에서도 D4에 보여야 합니다.
- 빈 셀은 빈 셀로 유지되어야 하며, 행과 열 위치가 밀리면 안 됩니다.

8. JavaScript 함수 구조

app.js는 기능별로 함수를 나누어 작성해 주세요.

반드시 다음 함수들을 포함해 주세요.

- `createGrid()`
  - 스프레드시트 그리드를 동적으로 생성합니다.
  - 열 헤더 A~I, 행 헤더 1~9, 입력 가능한 셀을 만듭니다.
  - 각 셀에 row, col 정보를 dataset으로 저장합니다.
  - 셀 클릭, focus, input 이벤트를 연결합니다.

- `updateFocusedCell(row, col)`
  - 현재 선택된 셀 좌표를 `Cell: D4` 형식으로 화면에 표시합니다.

- `highlightHeaders(row, col)`
  - 기존 하이라이트를 제거합니다.
  - 현재 선택된 셀의 열 헤더와 행 헤더를 하늘색으로 하이라이트합니다.
  - 현재 선택된 셀 자체도 시각적으로 표시합니다.

- `collectGridData()`
  - 모든 셀 input 값을 읽어서 2차원 배열로 반환합니다.
  - 빈 셀은 빈 문자열로 저장합니다.

- `exportSpreadsheet()`
  - `collectGridData()`를 호출합니다.
  - SheetJS를 사용해서 2차원 배열을 worksheet로 변환합니다.
  - workbook을 생성한 뒤 `Sheet1` 이름의 시트로 추가합니다.
  - `mini-spreadsheet.xlsx` 파일로 저장합니다.

9. 스타일 요구사항

- 전체 화면은 보기 좋고 단순한 과제용 UI로 구성해 주세요.
- 셀은 실제 스프레드시트처럼 격자 형태로 보여야 합니다.
- 열 헤더와 행 헤더는 일반 셀과 구분되게 배경색을 넣어 주세요.
- 현재 선택된 행/열 헤더는 하늘색으로 강조해 주세요.
- Export 버튼은 사용자가 쉽게 찾을 수 있는 위치에 배치해 주세요.
- 셀 입력창은 크기가 일정해야 하며 표가 흐트러지지 않게 해 주세요.

10. 테스트 기준

구현 후 다음 동작이 가능해야 합니다.

- 페이지 접속 시 A~I, 1~9 셀 그리드가 정상 표시됩니다.
- C1 셀을 클릭하면 `Cell: C1`이 표시됩니다.
- D4 셀을 클릭하면 D 열 헤더와 4 행 헤더가 하늘색으로 하이라이트됩니다.
- A2에 `3333`을 입력한 뒤 다른 셀을 클릭해도 값이 유지됩니다.
- D4에 한글 `ㅇ의`를 입력해도 값이 유지됩니다.
- `Export SpreadSheet` 버튼을 누르면 `.xlsx` 파일이 다운로드됩니다.
- 다운로드된 Excel 파일을 Google Spreadsheet에서 Import하면 웹앱에서 입력한 데이터와 같은 위치에 같은 값이 표시됩니다.

11. 코드 작성 방식

- HTML, CSS, JavaScript 코드를 각각 분리해서 제시해 주세요.
- 각 파일명을 명확히 표시해 주세요.
- 코드에는 초보자가 이해할 수 있도록 중요한 부분에 주석을 달아 주세요.
- 불필요하게 복잡한 프레임워크는 사용하지 말고 순수 HTML/CSS/JavaScript로 작성해 주세요.
- 서버, 데이터베이스, 로그인, Google API 연동은 구현하지 마세요.
