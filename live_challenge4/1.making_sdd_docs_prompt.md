2. SRS

Software Requirements Specification

미니 스프레드시트 웹앱 기능 명세서

1. 시스템 개요

본 시스템은 브라우저에서 실행되는 클라이언트 기반 미니 스프레드시트 애플리케이션이다. 사용자는 셀에 값을 입력하고, 현재 선택된 셀 정보를 확인하며, 전체 셀 데이터를 Excel 파일로 내보낼 수 있다.

2. 기능 요구사항

SRS-001. 그리드 생성

* 열은 A부터 I까지 생성한다.
* 행은 1부터 9까지 생성한다.
* 왼쪽에는 행 번호 헤더가 표시되어야 한다.
* 위쪽에는 열 문자 헤더가 표시되어야 한다.
* 각 셀은 사용자가 입력 가능한 요소여야 한다.

SRS-002. 셀 값 입력

* 각 셀은 독립적으로 값을 가져야 한다.
* 입력한 값은 다른 셀을 클릭하거나 이동해도 유지되어야 한다.
* 한글, 영어, 숫자 입력이 가능해야 한다.

SRS-003. 현재 선택 셀 좌표 표시

* 좌표는 열 문자와 행 번호를 조합하여 표시한다.
* 예: C열 1행 선택 시 Cell: C1
* D열 4행 선택 시 Cell: D4

SRS-004. 행/열 헤더 하이라이트

* 선택된 셀의 위쪽 열 헤더가 하늘색으로 표시된다.
* 선택된 셀의 왼쪽 행 헤더가 하늘색으로 표시된다.
* 새로운 셀을 선택하면 이전 하이라이트는 제거된다.

SRS-005. 셀 데이터 수집

* 셀 데이터는 2차원 배열 형태로 수집한다.
* 각 배열의 행은 스프레드시트의 행과 대응한다.
* 각 배열의 열은 스프레드시트의 열과 대응한다.
* 비어 있는 셀은 빈 문자열 ""로 저장한다.

SRS-006. Excel 파일 Export

* 버튼 이름은 Export SpreadSheet로 표시한다.
* 버튼 클릭 시 전체 셀 데이터를 수집한다.
* 수집된 2차원 배열을 Excel 워크시트로 변환한다.
* .xlsx 확장자의 파일을 다운로드한다.

SRS-007. Google Spreadsheet Import 호환성

* A1에 입력된 값은 Google Spreadsheet에서도 A1에 표시되어야 한다.
* D4에 입력된 값은 Google Spreadsheet에서도 D4에 표시되어야 한다.
* 빈 셀은 빈 셀로 유지되어야 한다.
* 행과 열의 상대적 위치가 변경되면 안 된다.
