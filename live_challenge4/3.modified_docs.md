1. PRD

Product Requirements Document

Mini Spreadsheet Web App

1. 제품 개요

1.1 제품명

Mini Spreadsheet Web App

1.2 제품 목적

이 프로젝트는 JavaScript 강의에서 학습한 DOM 조작, 이벤트 처리, 데이터 저장, 파일 생성 기능을 활용하여 만드는 과제용 웹 애플리케이션이다.

사용자는 웹 브라우저에서 간단한 스프레드시트 형태의 표를 사용할 수 있으며, 각 셀에 데이터를 입력하고, 선택한 셀의 위치를 확인하고, 행과 열을 추가 또는 삭제할 수 있다. 또한 입력한 데이터를 새로고침 후에도 유지할 수 있고, 최종 데이터를 .xlsx 또는 .csv 파일로 저장할 수 있다.

1.3 핵심 사용자

* JavaScript 기초 학습자
* 웹 프론트엔드 과제 수행자
* 간단한 스프레드시트 기능을 구현해보고 싶은 사용자

1.4 주요 기능 요약

2. 문제 정의

기존 HTML 표는 단순히 데이터를 보여주는 용도로 많이 사용된다. 하지만 이 프로젝트에서는 사용자가 직접 셀을 편집하고, 현재 선택된 셀을 확인하고, 데이터를 파일로 내보낼 수 있는 간단한 스프레드시트 기능이 필요하다.

또한 사용자가 브라우저를 새로고침해도 입력한 데이터가 사라지지 않아야 하며, 저장할 때는 실제 로컬 파일 저장 창을 통해 저장 위치, 파일명, 파일 형식을 선택할 수 있어야 한다.

4. 사용자 시나리오

시나리오 1: 셀 입력

1. 사용자가 웹페이지에 접속한다.
2. 기본 9행 × 9열 스프레드시트가 표시된다.
3. 사용자가 A2 셀에 3333을 입력한다.
4. 사용자가 D4 셀에 ㅇ의를 입력한다.
5. 다른 셀을 클릭해도 입력값은 유지된다.

시나리오 2: 셀 선택

1. 사용자가 C1 셀을 클릭한다.
2. 화면 상단에 Cell: C1이 표시된다.
3. C 열 헤더와 1 행 헤더가 하늘색으로 강조된다.
4. 사용자가 D4 셀을 클릭한다.
5. 화면 상단에 Cell: D4가 표시된다.
6. D 열 헤더와 4 행 헤더가 하늘색으로 강조된다.

시나리오 3: 새로고침 후 데이터 유지

1. 사용자가 여러 셀에 값을 입력한다.
2. 브라우저를 새로고침한다.
3. 기존에 입력한 값이 그대로 복원된다.
4. 추가하거나 삭제한 행/열 개수도 유지된다.

시나리오 4: 행/열 추가 및 삭제

1. 사용자가 Add Row 버튼을 클릭한다.
2. 마지막 행 아래에 새 행이 추가된다.
3. 사용자가 Add Column 버튼을 클릭한다.
4. 마지막 열 오른쪽에 새 열이 추가된다.
5. 특정 셀을 선택한 뒤 Delete Row를 누르면 해당 행이 삭제된다.
6. 특정 셀을 선택한 뒤 Delete Column을 누르면 해당 열이 삭제된다.

시나리오 5: 파일 Export

1. 사용자가 Export SpreadSheet 버튼을 클릭한다.
2. 로컬 파일 저장 창이 열린다.
3. 사용자가 저장 위치를 선택한다.
4. 파일명을 입력한다.
5. 파일 형식을 .xlsx 또는 .csv 중 선택한다.
6. 저장된 파일을 Excel 또는 Google Spreadsheet에서 열면 웹앱의 셀 위치와 동일하게 데이터가 표시된다.

2. SRS

Software Requirements Specification

Mini Spreadsheet Web App

1. 시스템 개요

Mini Spreadsheet Web App은 브라우저에서 실행되는 클라이언트 전용 웹 애플리케이션이다. 서버와 데이터베이스 없이 HTML, CSS, JavaScript만으로 구현한다.

시스템은 사용자가 셀에 입력한 데이터를 2차원 배열로 관리하며, 이 데이터를 localStorage에 저장한다. 사용자가 Export 버튼을 클릭하면 SheetJS를 사용해 데이터를 .xlsx 또는 .csv 파일로 변환하고, File System Access API를 통해 로컬 파일 저장 창을 표시한다.

⸻

2. 기능 요구사항

FR-001. 기본 스프레드시트 생성

설명

시스템은 페이지 로드 시 기본적으로 9행 × 9열의 스프레드시트 그리드를 생성해야 한다.

세부 요구사항

* 열 헤더는 A부터 I까지 표시한다.
* 행 헤더는 1부터 9까지 표시한다.
* 왼쪽 위에는 빈 코너 헤더를 둔다.
* 각 셀은 input type="text" 요소로 생성한다.
* 모든 셀은 JavaScript로 동적으로 생성한다.
* 각 셀은 dataset.row, dataset.col 값을 가져야 한다.

High

7.3 loadStateFromLocalStorage()

목적
