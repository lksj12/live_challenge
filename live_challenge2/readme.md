# Todos 앱

## 프로젝트 소개

이 프로젝트는 Vanilla JavaScript를 사용하여 구현한 Todo 웹 애플리케이션입니다.

정적인 HTML 화면을 사용자의 행동에 반응하는 앱으로 바꾸는 것을 목표로 하며, 사용자가 할 일을 추가하고, 완료 상태를 변경하고, 삭제하고, 수정할 수 있도록 구성했습니다.

이번 과제의 핵심은 React나 Shadcn 같은 프레임워크 없이 JavaScript의 기본 문법, DOM 조작, 이벤트 처리, 배열 데이터 관리, 화면 렌더링 흐름을 이해하는 것입니다.

## 핵심 문구

정적인 화면을 사용자의 행동에 반응하는 앱으로 바꾼다.

## 사용 기술

- HTML
- CSS
- Vanilla JavaScript
- LocalStorage

처음 목표는 정답 코드를 찾는 것이 아니라, Todo 앱을 만들기 위해 어떤 단계가 필요한지 이해하고 직접 구현 흐름을 따라가는 것이었습니다.

구현 순서는 다음과 같이 진행했습니다.

2. DOM 요소 선택하기
3. todos 배열 만들기

5. renderTodos 함수 만들기
6. 완료 및 삭제 기능 연결하기
7. 새로운 Todo 입력 박스 생성하기
8. Todo 수정 기능 추가하기

## 화면 구조

처음에는 화면을 기능별 영역으로 나누는 것부터 시작했습니다.

화면은 크게 다음 영역으로 구성했습니다.

- Header 영역
- Todo 입력 영역
- Todo 목록 영역
- 상태 표시 영역

이렇게 영역을 분리한 이유는 이후 JavaScript에서 각 영역을 선택하고 조작하기 쉽게 만들기 위해서입니다.

최종적으로 주요 HTML 구조는 다음과 같은 형태입니다.

- 새로운 TODO 추가하기 버튼
- Todo 목록이 렌더링되는 main 영역
- 남은 할 일 개수를 표시하는 상태 영역

## Todo 데이터 구조

Todo 하나는 단순 문자열이 아니라 객체 형태로 관리했습니다.

예시:

{
  id: Date.now(),
  text: "자바스크립트 이벤트 복습하기",
  completed: false
}

각 속성의 의미는 다음과 같습니다.

id:
Todo를 구분하기 위한 고유값입니다.

text:
사용자가 입력한 Todo 내용입니다.

completed:
Todo의 완료 여부를 나타내는 값입니다.
false이면 미완료, true이면 완료 상태입니다.

객체 형태로 Todo를 관리한 이유는 완료 상태 변경, 삭제, 수정 기능을 안정적으로 구현하기 위해서입니다.

## 주요 함수

이 프로젝트에서 사용한 주요 함수는 다음과 같습니다.

addTodo(text):
새로운 Todo 객체를 만들고 todos 배열에 추가합니다.

renderTodos():
todos 배열을 기준으로 화면에 Todo 목록을 다시 렌더링합니다.

toggleTodo(id):
특정 Todo의 completed 값을 반대로 바꾸어 완료 또는 미완료 상태를 변경합니다.

deleteTodo(id):
특정 id를 가진 Todo를 todos 배열에서 제거합니다.

updateCount():
완료되지 않은 Todo의 개수를 계산하여 화면에 표시합니다.

updateTodoText(id, text):
기존 Todo의 내용을 수정합니다.

saveTodos():
현재 todos 배열을 LocalStorage에 저장합니다.

renderInputBox():
새로운 Todo를 입력할 수 있는 입력 박스를 화면에 표시합니다.

## 핵심 동작 흐름

이 앱의 핵심 흐름은 다음과 같습니다.

사용자 행동 발생
→ todos 배열 데이터 변경
→ LocalStorage 저장
→ renderTodos() 호출
→ 화면 갱신
→ 남은 할 일 개수 갱신

즉, 화면 자체가 중심이 아니라 todos 배열이 중심입니다.

화면은 todos 배열을 렌더링한 결과입니다.

## 구현 과정 요약

### 1단계: HTML 구조 만들기

처음에는 index.html에서 화면을 영역별로 나누었습니다.

입력 영역, 목록 영역, 상태 영역을 분리하면 이후 JavaScript로 연결하기 쉽기 때문입니다.

초기 구조에서는 다음 요소들이 중요했습니다.

- 새로운 TODO 추가하기 버튼
- Todo 목록 영역
- 남은 할 일 개수 표시 영역

### 2단계: DOM 요소 선택하기

JavaScript에서 HTML 요소를 조작하기 위해 필요한 요소들을 선택했습니다.

선택한 주요 요소는 다음과 같습니다.

addTodoButton:
새로운 TODO 추가하기 버튼입니다.

todoList:
Todo 목록이 들어갈 영역입니다.

todoCount:
남은 할 일 개수를 표시할 영역입니다.

### 3단계: todos 배열 만들기

Todo 데이터를 저장하기 위해 todos 배열을 만들었습니다.

처음에는 빈 배열로 시작했습니다.

이후 LocalStorage 기능을 추가하면서, 저장된 데이터가 있으면 LocalStorage에서 불러오고 없으면 빈 배열로 시작하도록 변경했습니다.

### 4단계: addTodo 함수 만들기

addTodo(text) 함수는 입력받은 문자열을 Todo 객체로 변환한 뒤 todos 배열에 추가하는 역할을 합니다.

새로 추가된 Todo가 화면의 가장 위에 보이도록 push가 아니라 unshift를 사용했습니다.

### 5단계: renderTodos 함수 만들기

renderTodos() 함수는 todos 배열을 기준으로 화면을 다시 그리는 함수입니다.

처음에는 정적으로 작성된 Todo 예시가 있었지만, 이후에는 HTML에 직접 Todo를 작성하지 않고 JavaScript가 todos 배열을 기준으로 Todo item을 생성하도록 변경했습니다.

이 함수 안에서는 다음 작업을 수행합니다.

- 기존 Todo 목록 영역 비우기
- todos 배열을 순회하면서 Todo item 생성
- 완료된 Todo에는 completed 클래스를 추가
- 체크박스, 수정 버튼, 삭제 버튼 생성
- 각 버튼에 이벤트 연결
- 화면에 Todo item 추가
- 남은 할 일 개수 갱신

### 6단계: 완료 및 삭제 기능 연결하기

체크박스를 클릭하면 toggleTodo(id)가 실행되도록 했습니다.

toggleTodo(id)는 해당 Todo의 completed 값을 true에서 false로, 또는 false에서 true로 변경합니다.

삭제 버튼을 클릭하면 deleteTodo(id)가 실행되도록 했습니다.

deleteTodo(id)는 해당 Todo를 todos 배열에서 제거합니다.

두 함수 모두 데이터가 바뀐 뒤 renderTodos()를 다시 호출하여 화면을 갱신합니다.

## 추가 버튼 동작

처음에는 addTodo 함수를 콘솔에서 직접 호출하여 테스트했습니다.

이후 실제 사용자 흐름에 맞게 새로운 TODO 추가하기 버튼을 누르면 목록 맨 위에 입력 박스가 생성되도록 구현했습니다.

동작 흐름은 다음과 같습니다.

새로운 TODO 추가하기 클릭
→ 입력 박스 생성
→ 사용자가 할 일 입력
→ Enter 또는 펜 버튼 클릭
→ Todo 추가
→ 화면 다시 렌더링

입력 박스가 여러 개 동시에 생기지 않도록 isAdding 변수를 사용했습니다.

## 빈 값 방지

Todo를 추가하거나 수정할 때 빈 문자열이 저장되지 않도록 trim()을 사용했습니다.

입력값 앞뒤의 공백을 제거한 뒤 빈 값이면 Todo를 추가하지 않고 알림을 표시합니다.

## 수정 기능

펜 버튼을 클릭하면 해당 Todo가 수정 모드로 바뀌도록 구현했습니다.

수정 모드에서는 기존 텍스트가 span이 아니라 input으로 표시됩니다.

수정 완료는 다음 방식으로 처리했습니다.

- Enter 입력
- 펜 버튼 클릭

처음에는 다른 곳을 클릭해도 작성 완료로 처리하려고 했지만, 이후 사용성을 단순하게 하기 위해 blur 이벤트를 제거하고 Enter 또는 펜 버튼으로만 저장되도록 변경했습니다.

수정 중인 Todo를 구분하기 위해 editingId 변수를 사용했습니다.

editingId가 특정 Todo의 id와 같으면 해당 Todo는 수정 input 형태로 렌더링됩니다.

## LocalStorage 저장 기능

선택 기능으로 LocalStorage 저장 기능을 추가했습니다.

이 기능을 통해 새로고침해도 Todo 목록이 초기화되지 않습니다.

처음 페이지가 열릴 때는 LocalStorage에서 todos 데이터를 불러옵니다.

데이터가 없으면 빈 배열로 시작합니다.

todos 배열이 변경되는 상황마다 saveTodos()를 호출하여 LocalStorage에 저장합니다.

저장되는 경우는 다음과 같습니다.

- Todo 추가
- Todo 완료 상태 변경
- Todo 삭제
- Todo 내용 수정

## 대화 및 구현 과정 요약

처음에는 과제 요구사항을 이해하는 것부터 시작했습니다.

과제의 핵심은 JavaScript 기본기를 확인하는 것이며, React나 Shadcn 없이 Vanilla JavaScript 중심으로 구현해야 한다는 점을 확인했습니다.

그다음 HTML 구조를 먼저 작성했습니다.

처음에는 입력 영역, 목록 영역, 상태 영역을 단순히 나누었지만, 이후 목표 UI 이미지를 기준으로 더 구체적인 Todo 카드 형태의 화면을 만들었습니다.

이후 사용자가 지적한 것처럼 새로 추가되는 Todo 입력 박스가 가장 위에 나와야 하기 때문에, 입력 중인 Todo 영역이 목록 맨 위에 오도록 구조를 조정했습니다.

펜 아이콘은 처음에는 이모지로 사용했지만, 브라우저마다 다르게 보일 수 있으므로 SVG 펜 아이콘으로 변경했습니다.

CSS는 처음에는 index.html 내부 style 태그에 작성했지만, 이후 style.css 파일로 분리했습니다.

JavaScript 구현은 단계적으로 진행했습니다.

먼저 DOM 요소를 선택했고, todos 배열을 만들었습니다.

그다음 addTodo(text) 함수를 만들어 콘솔에서 직접 Todo가 배열에 들어가는지 테스트했습니다.

이후 renderTodos() 함수를 만들어 todos 배열에 있는 데이터가 화면에 표시되도록 했습니다.

남은 할 일 개수가 바뀌지 않는 문제를 확인한 뒤 updateCount() 함수를 추가했습니다.

그다음 체크박스 완료 처리와 삭제 버튼 기능을 연결했습니다.

이후 새로운 TODO 추가하기 버튼을 눌렀을 때 입력 박스가 생성되도록 구현했습니다.

처음에는 입력 박스나 Todo가 사라지는 문제가 있었는데, 원인은 renderTodos()가 todoList.innerHTML = ""로 화면을 다시 비우기 때문이었습니다.

이를 해결하기 위해 HTML에 정적 Todo를 남겨두지 않고, todos 배열과 isAdding 상태를 기준으로 renderTodos()가 화면 전체를 다시 그리도록 구조를 정리했습니다.

그다음 펜 버튼을 눌렀을 때 수정이 되지 않는 문제를 확인했고, editingId를 사용하여 수정 모드를 구현했습니다.

한때 다른 곳을 클릭해도 저장되도록 blur 이벤트를 추가했지만, 이후 요구사항에 맞게 blur 저장은 제거하고 Enter 또는 펜 버튼 클릭으로만 저장되도록 수정했습니다.

마지막으로 LocalStorage 저장 기능을 추가하여 새로고침해도 Todo 목록이 유지되도록 만들었습니다.

## 배운 점

이 프로젝트를 통해 가장 중요한 점은 화면을 직접 기준으로 생각하는 것이 아니라, 데이터 배열을 기준으로 화면을 다시 그리는 방식입니다.

즉, Todo 앱의 중심은 HTML 요소가 아니라 todos 배열입니다.

사용자가 어떤 행동을 하면 todos 배열이 변경되고, 변경된 배열을 기준으로 renderTodos()가 화면을 다시 그립니다.

이 흐름을 이해하면 추가, 삭제, 완료 변경, 수정, 저장 기능을 모두 같은 패턴으로 구현할 수 있습니다.

## 핵심 정리

Todo 앱의 중심은 화면이 아니라 todos 배열이다.

데이터가 바뀌면 renderTodos()를 다시 호출한다.

renderTodos()는 현재 todos 배열 상태를 기준으로 화면을 새로 그린다.

LocalStorage는 todos 배열을 문자열로 저장하고, 다시 객체 배열로 불러오는 역할을 한다.

이번 과제는 JavaScript의 기본 구조를 이해하고, DOM 조작과 이벤트 처리를 연습하는 데 목적이 있다.
