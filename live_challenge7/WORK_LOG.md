# AI·에이전트 작업 기록

## 1. 최초 Implementation Plan

- 계획 요약:
  - 과제 안내와 Starter 문서를 분석합니다.
  - React·TypeScript·Redux 기반 구조를 먼저 구성합니다.
  - 프론트엔드는 `client/`, 백엔드는 `server/`로 분리합니다.
  - 기본 CRUD 이후 인증, 저장, 도전 기능을 작은 단계로 구현합니다.
  - 각 단계마다 typecheck, lint, test, build를 실행합니다.
- 사람이 수정한 내용:
  - Google Antigravity IDE 대신 Codex를 사용하도록 변경했습니다.
  - 외부 DB 대신 SQLite를 사용하도록 결정했습니다.
  - 관리자·일반 사용자 역할과 서버 세션 revoke를 추가했습니다.
  - 비회원 생성·삭제는 허용하되 태그는 제한하고 저장 경고를 요구했습니다.
  - 로그인 화면 문구와 반응형 줄바꿈을 직접 검토했습니다.
- 승인 결과:
  - 사용자가 단계별로 `다음`, `ㄱ`, `ㄱㄱ`를 전달한 뒤 해당 작은 작업을 진행했습니다.

## 2. 전체 초안 검토

- 구현된 기능:
  - 회원·비회원 노트 CRUD
  - 검색, 정렬, 태그, 고정, 보관함, 휴지통
  - 색상·우선순위·Markdown·자동 저장·다크 모드
  - 인증·세션 revoke·SQLite·관리자 화면
  - 로딩·빈 상태·오류·접근성 처리
- 실제 변경 파일:
  - `client/src/`: React 화면, API, Redux, 저장소, 타입, 테스트
  - `server/src/`: Express API, 인증, SQLite, 관리자, 테스트
  - `AGENT_GUIDE.md`, `.agents/rules/project.md`
  - `README.md`, `WORK_LOG.md`, `docs/plan.md`, `docs/spec.md`
- build 결과:
  - `npm run typecheck`: 성공
  - `npm run lint`: 성공
  - `npm run test`: 클라이언트 27개·서버 16개, 총 43개 성공
  - `npm run build`: 성공
- 화면 테스트 결과:
  - 개발 서버와 API health 응답을 확인했습니다.
  - 기능 구현 중 제공된 화면과 사용자 피드백을 반영했습니다.
  - 데스크톱·모바일 전체 수동 시나리오와 콘솔 전수 확인은 제출 전 남아 있습니다.
- 발견한 문제:
  - 루트가 아직 Git 저장소가 아니어서 diff·commit·저장소 URL 증거가 없습니다.
  - HMR 중 리듀서 구조 변경으로 기존 페이지 상태가 일시적으로 깨져 서버를 재시작했습니다.

## 3. 수정 작업

| 문제 | 원인 | 수정 내용 | 재검증 결과 |
|---|---|---|---|
| 비회원 저장 정책 충돌 | LocalStorage는 브라우저 종료 후에도 남음 | 노트 저장을 `sessionStorage`로 변경 | 스토리지 테스트 통과 |
| 경고 취소 시 입력 손실 위험 | 경고와 편집기 닫기 상태가 결합됨 | 중첩 대화상자 상태를 분리 | 입력 상태 유지 |
| 로그아웃 토큰 재사용 위험 | 쿠키 삭제만으로 서버 세션이 남을 수 있음 | 토큰 해시 세션에 `revoked_at` 기록 | 기존 토큰 401 테스트 통과 |
| 사용자 간 데이터 접근 위험 | 클라이언트 ID를 신뢰할 가능성 | 서버 세션 사용자 ID로 모든 쿼리 제한 | 타 사용자 접근 404 테스트 통과 |
| 태그 소유권 혼합 위험 | 임의 태그 ID 지정 가능 | 태그와 노트 소유자를 서버에서 함께 검증 | 타 사용자 태그 400 테스트 통과 |
| 자동 저장 중복 생성 위험 | 신규 노트 ID가 편집기 props에 없음 | 첫 저장 ID를 `workingNoteId`로 유지 | 이후 PATCH 경로 사용 |
| effect 린트 오류 | effect 본문에서 대기 상태를 즉시 변경 | 실제 요청 시작 시에만 상태 변경 | lint 통과 |
| Markdown 테스트 실패 | `skipHtml`이 HTML 블록 전체 제거 | 안전 문단을 별도 블록으로 검증 | 테스트 통과 |
| 다크 모드 HMR 오류 | 기존 store에 새 slice가 없음 | 개발 서버 완전 재시작 | 런타임 정상 |
| 모달 키보드 이탈 | 공통 포커스 관리 없음 | 포커스 트랩·복귀 훅 적용 | typecheck·lint·build 통과 |
| 자유 색상 형식 오류 | 서버는 6자리 HEX만 허용 | 3·6자리 정규화와 입력 오류 표시 | 색상 테스트 통과 |

## 4. 사람의 최종 판단

- 요구사항 충족 여부:
  - React·TypeScript·Redux Toolkit 필수 기술과 노트 기본 기능을 충족했습니다.
  - 합의한 SQLite, 인증, 비회원, 관리자, 도전 기능을 코드와 테스트에 반영했습니다.
- 남은 문제:
  - Git 저장소 URL과 commit 기록이 없습니다.
  - 브라우저 전체 수동 시나리오·콘솔·화면 캡처가 남아 있습니다.
- 다음 개선점:
  - Git 저장소 초기화와 제출 URL 연결
  - 데스크톱·모바일·라이트·다크 수동 검증
  - 제출 화면 캡처와 최종 diff 검토

최종 채택 문장:

> 자동화 검증을 통과한 현재 구현을 제출 후보로 채택하되, 저장소 URL과
> 브라우저 전수 검증 증거를 추가한 뒤 최종 제출합니다.
