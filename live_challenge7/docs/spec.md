# React 노트 앱 통합 스펙

## 1. 공식 확인 요구사항

- React를 이용하여 노트 앱을 제작합니다.
- Redux와 같은 상태관리 라이브러리를 사용합니다.
- TypeScript를 사용합니다.
- 결과물이 올라간 저장소 주소를 제출합니다.
- 제공된 결과 예시와 과제 안내를 참고합니다.

> 공식 과제 안내와 이 문서가 충돌하면 공식 과제 안내를 우선합니다.

## 2. 프로젝트 확정 기능

### 노트

- 노트 목록·작성·확인·수정·삭제
- 빈 상태와 공백 제목 차단
- 검색, 수정일·생성일·우선순위 정렬
- 색상, 우선순위, 고정 노트
- 태그 생성·복수 지정·필터
- 보관함·휴지통·복원·영구 삭제
- Markdown 미리보기
- 자동 저장
- 라이트·다크 모드

### 인증과 저장

- 회원가입·로그인·로그아웃
- 관리자·일반 사용자 역할 분리
- 로그인 토큰의 서버 세션 revoke
- 회원 노트 SQLite 저장
- 비회원 노트 `sessionStorage` 저장
- 비회원 생성·수정·삭제
- 비회원 태그 생성·지정 제한과 로그인 안내
- 신규 비회원 저장 전 데이터 손실 가능성 확인

### 관리자

- 일반 사용자 목록
- 사용자별 노트 개수와 저장 용량
- 최근 로그인 조회
- 비밀번호 초기화와 기존 세션 revoke
- 사용자 로그인 정보와 관련 데이터 삭제
- 관리자 작업 감사 로그

### 품질

- 로딩·오류·빈 결과 상태
- 반응형 데스크톱·모바일 UI
- 모달 포커스 트랩·복귀
- 키보드 탭과 상태 안내
- 자동 테스트

## 3. 데이터 구조

```ts
export interface Note {
  id: string
  ownerId: string | null
  title: string
  content: string
  color: string
  priority: 'low' | 'medium' | 'high'
  status: 'active' | 'archived' | 'trashed'
  isPinned: boolean
  tags: string[]
  sizeBytes: number
  createdAt: string
  updatedAt: string
}
```

SQLite 주요 테이블:

- `users`: 로그인 정보, 역할, 상태
- `sessions`: 해시된 토큰, 만료와 revoke
- `notes`: 사용자 노트
- `tags`: 사용자 태그
- `note_tags`: 노트와 태그 관계
- `audit_logs`: 관리자 작업 기록

## 4. 상태관리 기준

### Redux 전역 상태

- 인증 사용자와 요청 상태
- 회원·비회원 노트 목록
- 검색·정렬·태그·목록 상태
- 태그 목록
- 관리자 사용자 통계
- 라이트·다크 테마

### React 로컬 상태

- 작성 중 제목·내용·색상 코드
- 열린 대화상자
- Markdown 작성·미리보기 탭
- 비밀번호 표시와 확인 입력
- 자동 저장 표시 상태

저장된 노트 원본은 Redux를 기준으로 하며 로컬 상태와 중복 보관하지 않습니다.

## 5. 저장·보안 기준

- 비밀번호는 Argon2id 해시만 SQLite에 저장합니다.
- 로그인 토큰은 HttpOnly 쿠키에 두고 서버에는 SHA-256 해시만 저장합니다.
- 로그아웃과 관리자 비밀번호 초기화는 관련 세션을 revoke합니다.
- 서버는 클라이언트가 전달한 사용자 ID와 역할을 신뢰하지 않습니다.
- 노트와 태그 쿼리는 인증된 세션 사용자 ID로 제한합니다.
- 비회원 노트는 서버 API로 보내지 않습니다.
- Markdown 원본 HTML은 렌더링하지 않습니다.
- 실제 개인정보·DB 파일·비밀번호를 저장소에 포함하지 않습니다.
